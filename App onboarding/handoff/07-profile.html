Phase 7

Profile tab

Identity card + the full settings tree — a root hub and seven push-in sub-screens, plus four confirm sheets. Three settings here are the canonical writers of cross-tab state (theme, week-start, log-stat) that the rest of the app reads.

Sourcecombined/profile.jsx (315 lines — window.ProfileTab). Reuses Toggle, Seg, SetRow, Stepper from ui.jsx. Receives tab, onTab, dark, onDark, calStart, onCalStart, logStat, onLogStat from AuraRoot — it gets every setter (see Ch.2).

Structure & root

A screen string acts as a mini-router: null shows the root hub; any of the seven names pushes a full-screen settings page (each with a back-to-Profile header). This is the same early-return pattern as the Plan tab.

Root hub

  • Identity card (tap → Account): gradient initials avatar, "{first} {last}", "28 · {height} cm · 78.4 kg · {gender}".
  • Three stat tiles: Sessions 142 · PRs 37 · Week streak 18.
  • Three setting groups of rows → push screens: General · Workout · Notifications / Account Details · Units & Measurements · Connected Apps / Support. The Units & Connected rows show live sub-text (current units / "Apple Health connected").
  • Log Out (red) → logout confirm sheet.

State & cross-tab sync

  • cfg holds every setting (calStart, logStat, showFirst, showPR, defSets, defRepsLo/Hi, restSet, restEx, autoRest, autoVideo, notif, restSound, weightUnit, lengthUnit, apple, google). acct holds the account fields.
  • The sync bridge: set(k,v) updates cfg and — for calStart and logStat — also calls onCalStart/onLogStat up to the shell, which persists them and feeds Log/Progress. Theme is handled directly via onDark. These three are the only settings that escape Profile; everything else is local cfg in the prototype.
NativeIn the prototype only theme / week-start / log-stat are wired through to other tabs and persisted; the rest of cfg (rest defaults, units, etc.) is in-memory. In the real build, all of these should persist and actually drive behaviour — e.g. defSets/restSet should seed the Active Workout, weightUnit should convert displays.

The seven settings screens

SET · general

General

  • Dark Mode toggle → onDark ("Applies across the app").
  • Start week on — Sun/Mon segmented → onCalStart (drives Log + Plan week strips).
  • Show on progress — Strength score / Balance / Both segmented → onLogStat (gates the Progress Performance cards, Ch.6).
SET · workout

Workout

  • Display: "Show first" Reps·time / Weight segmented; "Show PRs during workout" toggle.
  • Exercise targets: Default sets stepper (1–10); Default rep range — two interlocked steppers (lo max-capped by hi, hi min-capped by lo); Rest between sets & Rest between exercises steppers (15–300s, step 15, formatted "1 min 30 s").
  • Automation: Auto rest timer, Auto-play video toggles.
SET · account

Account Details

  • Avatar + Change photo (stub toast). Editable fields: First/Last, Email, Phone, Birthday (date), Gender (select), Height, Country/City/State.
  • Data: Export Data → export sheet; Reset Data → reset sheet. Delete Account (red) → delete sheet. Save Changes returns to root + flashes.
SET · notifications

Notifications

  • Enable notifications toggle. Rest timer sound list (Ding / Alarm clock, check-marked) — the whole sound list is dimmed + non-interactive when notifications are off.
SET · units

Units & Measurements

  • Weight unit (Kilograms / Pounds) and Length unit (Centimeters / Inches) segmented. The root hub's Units row reflects the choice live.
SET · connected

Connected Apps

  • Apple Health + Google Health toggles with "Connected / Not connected" sub-text. Info card notes two-way sync of workouts + body weight.
SET · support

Support

  • User Guides & FAQ · Contact Us · Feature Request (all stub toasts). Version footer "Aura Fitness · v2.4.0".

The four confirm sheets

kindWhat
exportExport Data — CSV + JSON archive; "Export Archive" flashes.
resetReset Data — "workout data only" (keeps profile) or "reset everything" (red).
deleteDelete account? — red destructive confirm ("cannot be undone").
logoutLog out? — primary confirm ("log back in anytime"). Shares the sheet renderer with delete.
FABOn Profile, workout → Log; measure/photo → Progress (see Ch.2 routing).

Scenario checklist

  • Identity card + Account row both open Account Details; Save returns to root.
  • Dark Mode, Start week, Show-on-progress in General immediately affect the rest of the app and persist (they route through the shell setters).
  • Workout: rep-range steppers stay valid (lo ≤ hi); rest steppers format as "min/s".
  • Notifications: turning off notifications dims + disables the sound list; sound selection check-marks.
  • Units row sub-text on the hub matches the chosen weight/length units.
  • Connected toggles flip the "Connected/Not connected" sub-text; hub Connected row reflects Apple state.
  • Export / Reset (two options) / Delete / Log Out confirm sheets open and flash their outcomes.
  • Every sub-screen has a working back-to-Profile header.
