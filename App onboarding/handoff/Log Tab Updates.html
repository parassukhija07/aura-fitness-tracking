Log Tab
Design Updates

This document covers all design changes to the Log tab — new screens, updated behavior rules, and UX fixes. Hand this to Claude Code alongside the reference HTML to implement natively in SwiftUI.

The HTML files are design references, not production code. Recreate these screens in the existing SwiftUI codebase using its established patterns, components, and navigation primitives. Do not ship the HTML directly.

Change summary

LOG-01
Exercise list — scrollable, all exercises visible

The static "+ N more exercises" truncation is replaced with a fixed-height scrollable list showing every exercise in the workout card.

LOG-01b
⋯ menu sheet — new screen

Bottom sheet triggered by the ellipsis button on today's planned workout card. Five action rows, each with a subtitle clarifying scope (today only).

LOG-01c
Log a Past Workout sheet — new screen

Two-section bottom sheet: pick a date (quick options + calendar), then pick a workout source.

LOG-01d
Past date · two states — new screens

Completed: green badge + checkmark, View Log + Edit Log CTAs. Missed: red badge + X, dimmed exercises, Log This Workout + Mark as Rest Day CTAs.

LOG-01e
Future date · locked — new screen

Program-assigned workout shown read-only. "View only" badge replaces ⋯. Single View Workout CTA. Info banner: "Future workouts are read-only — edit in Plans."

LOG-02
Rest day — removed duplicate CTA

"Convert to training day" row removed. The single "Add a Workout" button is sufficient — converting the plan structure belongs in the Plan tab.

LOG-01b + LOG-06
Today-only scope clarification

All scheduling action rows in the ⋯ menu and the Manage Today sheet now carry subtitles confirming actions affect today only and don't modify the underlying program.

Detailed specs

Updated LOG-01 — Exercise list

The exercise list inside the planned workout card is now a fixed-height scrollable container.

Implementation

  • Container: max-height: 162pt, overflow-y: scroll, clipped
  • Show all exercises — no truncation, no "+ N more" label
  • Each row: number badge · exercise name · sets + reps sub-label · chevron
  • Card outer dimensions are unchanged — only the internal list is now scrollable
  • Use native ScrollView (SwiftUI) with .frame(maxHeight: 162)

New LOG-01b — ⋯ menu sheet

Tapping the ellipsis button on the planned workout card presents a detent sheet. Background content blurs behind the sheet.

Sheet header

  • Workout name centered, bold 15pt
  • Subtitle: "Planned for today" — muted, 13pt

New LOG-01c — Log a Past Workout

Triggered by the "Log past" secondary button on LOG-01. A two-section bottom sheet.

Section 1 — When did you train?

  • Three grouped rows in a rounded list card:
  • Yesterday with sub-label showing the actual date (e.g. "Sun, Jun 21") — default selected (filled accent radio)
  • 2 days ago with sub-label date — unselected (empty circle)
  • Pick a date — navigates to system date picker, chevron

Section 2 — Which workout?

  • Three source cards identical in style to LOG-05 source picker:
  • From a Program — accent icon, "Your active PPL plan & saved programs"
  • A Saved Workout — blue icon, "Custom & predefined workouts"
  • Build from Library — green icon, "Pick exercises from scratch"
  • No "Empty Workout" option — logging a past session implies a known workout

State

  • Store selected date in local sheet state; pass to workout source flow on selection

New LOG-01d — Past date · two states

Shown when the user navigates to any past date in the week bar. The workout was assigned by the active program. Two distinct states depending on completion.

State A — Completed

  • Week bar: selected day shows a green done dot. "‹ Prev week" button visible.
  • Badges row: Program name badge (accent) + "Completed" badge (green fill, check icon)
  • Card header: workout name + muscle groups sub-label + green circle with check-circle icon (right side, replacing ⋯)
  • Exercise list: scrollable, full opacity, chevrons visible
  • CTAs:
    • View Log — primary button — opens the logged set data for that session
    • Edit Log — secondary gray button — allows editing logged weights/reps/sets retroactively
  • No Start Workout button. No Switch button. No ellipsis menu.

State B — Missed

  • Week bar: selected day shows a red/muted dot (distinct from done/plan/rest)
  • Badges row: Program name badge (accent) + "Missed" badge (red fill, X icon)
  • Card header: workout name + muscle groups sub-label + red circle with X-circle icon (right side)
  • Exercise list: scrollable, opacity: 0.55, no chevrons
  • CTAs:
    • Log This Workout — primary button — opens LOG-01c (log past) pre-filled with this date
    • Mark as Rest Day — secondary gray button — reclassifies this day and clears the missed state
  • No Edit Log button. User must first log, then they can edit.

New LOG-01e — Future date · locked

Shown when the user navigates to any future date in the week bar that has a program-assigned workout.

  • Week bar: selected day shows a plan dot (orange). "Today ›" button visible to jump back.
  • Badge: Program name badge (accent) only — no status badge
  • Card header: workout name + muscle groups sub-label + "View only" text badge (muted fill, text-3 color) replacing ⋯
  • Exercise list: scrollable, opacity: 0.55, no chevrons (read-only feel)
  • CTAs:
    • View Workout — gray (not primary) button — opens a read-only detail view of the planned exercises
    • No Start, No Log past, No Edit, No Switch buttons.
  • Info banner below CTA:
    • Accent-tinted background, accent border, info icon
    • Title: "Future workouts are read-only" — 14pt bold
    • Body: "To change this workout, edit your program in the Plans tab." — 13pt muted
    • Ghost link: "Go to Plans →" — accent color, navigates to Plan tab

Fix LOG-02 — Rest day

  • Remove the "Convert to training day" list row entirely
  • Keep only: section header "Did you train anyway?" + full-width "Add a Workout" tinted button
  • Rationale: both CTAs led to the same outcome; converting plan structure belongs in Plan tab

Updated LOG-01b + LOG-06 — Today-only scope subtitles

  • Any action that changes the day's workout (switch, edit, move, remove) must carry a subtitle: "Today only · your program stays unchanged" or equivalent phrasing
  • LOG-06 bottom of sheet: add a small muted caption — "All changes apply to today only and won't affect your program."
  • This rule applies everywhere in the Log tab. Never expose a program-editing action from the Log tab.

State model additions

DayState enumAdd cases: .completedWorkout, .missedWorkout, .futureWorkout. Derive from date comparison + workout log existence.
isEditableBool — true only for past completed days. Gates "Edit Log" button visibility.
isFutureBool — derived from date > Calendar.current.startOfDay(for: .now). Gates all write actions from Log tab.
selectedLogDateCarried from LOG-01c date picker into the workout source picker flow so the session is stamped with the correct past date.
todayOnlyActionsSwitch, Move, Edit from the ⋯ menu must operate on a copy of today's scheduled workout, not mutate the program's recurring template.
