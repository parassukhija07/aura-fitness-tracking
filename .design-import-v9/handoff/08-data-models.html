Phase 8

Data models & seed reference

Every state shape, seed dataset and computed-value formula in one place. The per-tab chapters reference these; this is the single sheet to keep open while wiring real data into the native models.

ReminderAll shapes below are the prototype's in-memory React state. Only the keys in localStorage persist; everything else resets on reload. The native build must back these with a real store (and persist far more than the prototype does — workout logs, overrides, all of cfg).

Log model (combined/log.jsx)

  • State: selected (Date), overrides ({iso → override}), sheet, calMonth, toast, workoutLogs ({iso → {time, exercises[]}}).
  • Override = { type, workoutId?, exercises? } — type ∈ rest · removed · added · logged · edit · switch.
  • Seed: PLAN_BY_DOW {Sun null, Mon pull-a, Tue leg-a, Wed null, Thu push-a, Fri pull-b, Sat null}; SEED_MISSED {2026-06-23, -06-17, -06-11}; WK_DETAIL (workout → {name, muscles, duration, ex:[[name,sets,reps]]}); OTHER_PLANS (HIIT, 5×5).
  • workoutLog set = { weight, reps } (strings).

The derived kind (today · done · missed · future · rest · empty-today) is computed by dayInfo() — full table in Ch.3.

Plan seeds (plan/data.jsx)

  • PLAN_WORKOUTS (5): { id, name, exCount, muscles, duration }.
  • PLAN_PROGRAMS (6): { id, name, days, level, tag, active } — ppl active.
  • PLAN_EXERCISES_LIB (47): { id, name, muscle, equip } — 7 muscles × 6 equipment.
  • DEFAULT_SCHEDULE: {Mon push-a, Tue pull-a, Wed null, Thu leg-a, Fri push-b, Sat/Sun null}.
  • Editor shape (WK_SEEDS): exercise = { id, name, sets, reps, superset } — superset flag on the leader; partner = next item.

Progress data (combined/progress.jsx)

  • measure = 9 keys {weight, bodyFat, chest, waist, arms, thighs, shoulders, neck, hips}; series = {key → 8-pt array} (capped 8 on log).
  • nut = { height:178, weight:78.4, age:28, sex:'Male', activity:'Moderate', target:80, goal:'Lean gain', macro:'Balanced' }.
  • Seeds: PR_DATA (per-category records), EX_DATA (generated 12-pt trend series), SPLIT, WEEKLY_VOL/SETS, heatmap via seeded PRNG genMonth.
  • Constants: ACT (activity multipliers), GOAL_ADJ (calorie deltas), MACRO_SPLIT (%[P,C,F]).

Profile cfg (combined/profile.jsx)

  • cfg: calStart, logStat, showFirst, showWeightFirst, showPR, defSets:3, defRepsLo:6, defRepsHi:10, restSet:60, restEx:90, autoRest, autoVideo, notif, restSound:'Ding', weightUnit:'kg', lengthUnit:'cm', apple:true, google:false.
  • acct: first, last, email, phone, bday, gender, height:178, country, city, state.
  • Only calStart/logStat/theme propagate to the shell & persist; the rest is local (should persist + drive behaviour natively).
