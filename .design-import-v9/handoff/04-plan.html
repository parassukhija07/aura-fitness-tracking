Phase 4

Plan tab

The library and authoring surface. Four sub-tabs (My Plans, Programs, Workouts, Exercises) sit under a shared header, and four full-screen views push over the whole tab for detail and editing — program detail, program editor, workout editor (with drag-reorder + supersets) and a rich three-tab exercise detail. This is the largest chapter by far.

Sourceplan/app.jsx (1258 lines — window.PlanApp), plan/data.jsx (seed: workouts, programs, exercise library, schedule), plan/exercise-detail.jsx (window.ExerciseDetail + body-map + history/PB math). Receives tab, onTab, calStart from AuraRoot.

Overview & navigation

The tab renders the phone shell (status bar, "Plan" large title + + button, sub-tab pill row) and one of four sub-tab bodies. Detail/editor screens are not sheets — they are full-screen views swapped in by early returns at the top of App, before the normal shell renders.

Navigation patternFive pieces of state act like a tiny router, checked in priority order: viewingEx → editingWk → editingProg → viewingProg → (else the sub-tab shell). Whichever is set wins and replaces the entire tab (its own back button clears it). There is no animated push — it's a hard swap. Map these to real navigation pushes in the native build.

Plan shell→ viewingProg (Program detail)→ editingProg (Program editor)→ editingWk (Workout editor)→ viewingEx (Exercise detail)

Persistence caveatEvery "Save" button in the editors simply calls onBack — the prototype does not persist editor changes back into the seed/program. In the native build, wire these saves to real mutations. Treat the editors as fully-specced UIs whose commit path is stubbed.

Seed data (plan/data.jsx)

  • PLAN_WORKOUTS (5): push-a, pull-a, leg-a, push-b, pull-b — each { id, name, exCount, muscles, duration }.
  • PLAN_PROGRAMS (6): ppl (active) · ul · arnold · fb3 · phul · bro — { id, name, days, level, tag, active }.
  • PLAN_EXERCISES_LIB (47): the master exercise catalog, { id, name, muscle, equip }, across 6 muscles (Chest, Back, Shoulders, Biceps, Triceps, Legs, Core) and 6 equipment types (Barbell, Dumbbell, Cable, Machine, Bodyweight, Smith).
  • DEFAULT_SCHEDULE: Mon push-a · Tue pull-a · Wed rest · Thu leg-a · Fri push-b · Sat/Sun rest.
  • Editor-only seeds in app.jsx: WK_SEEDS (per-workout exercise lists with {id,name,sets,reps,superset}) and WK_ADD_OPTIONS.

Colour / icon heuristics

Keyword-driven themingWorkout cards/strip tiles are coloured by a name keyword match, not stored data. wkStyle(w) / wkIcon(w) inspect the lowercased name: push → accent + flame, pull → blue + bolt, leg → green + trophy, upper → purple + arrow-up, else accent + dumbbell. A custom workout named "Chest Day" therefore falls to the default accent/dumbbell. Preserve this mapping (or replace with an explicit per-workout colour/icon field — better — but then port the defaults).

Sub-tab 1 · My Plans

PLAN · myplans

Active plan + week schedule + workouts

Plan carousel
  • Horizontal scroll of gradient plan cards: Push Pull Legs (with white "Active" badge), Upper / Lower, Full Body 3×, then a dashed "New" tile → opens the Add-plan sheet.
Workouts in program (MyPlansView)
  • List of the active plan's workouts: coloured icon tile (keyword), name, muscle list (split on ", "), and per-row edit (→ workout editor) + delete (red trash) buttons.
  • Section header has a + → Add-workout sheet. Deleting a workout also nulls any schedule day pointing at it.
PLAN · week-strip

Weekly schedule strip

Seven day tiles (WeekStrip) showing which workout is on each day. Shared with the Program editor.

  • Day order follows calStart: Mon-first by default, Sun-first when calStart==='Sun'.
  • Training day: keyword-coloured tile with the workout icon + short name. Tapping → Day menu.
  • Rest day: outlined tile with a moon glyph + "Rest". Tapping → Assign sheet to put a workout on that day.

Sub-tab 2 · Programs library

PLAN · programs

Browse & filter programs

  • Search box (matches name / tag / level) + a row of 5 filter dropdowns: Frequency · Equipment · Level · Split · Type. Each opens its own bottom-sheet option list (with "All" to clear); a red Clear chip appears when any filter is active and active filters tint accent.
  • Filter values come from a side PROG_META table keyed by program id (frequency/equipment/split/type), except Level which reads the card's own level. Empty result → "No programs found".
  • Program cards: thumb, name, "{freq} · {level} · {tag}". Active program shows an accent "Added" badge; others a chevron. Tap → Program detail.

Sub-tab 3 · Workouts library

PLAN · workouts

Browse workouts

  • Search + chip filters All · Push · Pull · Legs · Upper · Chest · Back (matches against name or muscles). Empty → "No workouts found".
  • Library cards (thumb, name, "{exCount} exercises · {muscles}") → tap opens the Workout editor.

Sub-tab 4 · Exercises library

PLAN · exercises

Browse the exercise catalog

  • Search + two filter rows: muscle chips (colour-coded per muscle) and equipment chips. The Arms muscle chip maps to Biceps + Triceps (those are stored separately but labelled "Arms").
  • 2-up catalog grid (cat-item): muscle-tinted gradient thumb with the muscle name, exercise name, "{muscle} · {equip}". Tap → Exercise detail with showActions.

The five plan sheets

SHEET · add-plan

Add to My Plans

  • Three sources: Browse programs → switches to the Programs sub-tab; Build from scratch → opens the Program editor; Duplicate active plan → (copy PPL) closes.
  • Opened by the header +, the carousel "New" tile, or the carousel +.
SHEET · assign

Assign workout to a day

  • Lists the plan's workouts; the currently-assigned one is highlighted with an accent ring + check. Picking one writes schedule[day]=id.
  • Footer: Create new workout source card, and Keep as Rest Day (closes without assigning).
SHEET · day-menu

Day actions

  • For a training day: Edit workout (→ workout editor), Change workout (→ assign sheet), Make it a rest day, and Remove from program (red). Both rest/remove null the day.
SHEET · add-workout

Add / create a workout

  • add-workout: "From Workout Library" (→ Workouts sub-tab) or "Create custom workout" (→ create-workout sheet).
  • create-workout: name field + a 12-icon picker grid (Push/Pull/Legs/Upper/Weights/Full Body/Core/Strength/Cardio/Cable/Hypertrophy/Recovery). "Continue → Add Exercises" creates the workout (disabled until named) and immediately opens the workout editor on it.

Predefined program preview

Full-screen · Program editor (build from scratch)

VIEW · program-editor

Create a custom weekly program

  • Name (auto-focused large input) and an optional Difficulty segmented control (Beginner green / Intermediate accent / Advanced red, toggle to clear).
  • Week strip (shared WeekStrip): tapping a day tries to assign the first workout. With no workouts yet, it instead flashes an accent warning ("Add workouts below before assigning days") for 2.5s — you must add workouts first.
  • Workouts section: empty-state offers "Add from Workout Library" or "Create your own workout"; once populated, each is an editable-name row with chevron (→ editor) + delete. A + opens the add sheet (library list or create-own).
  • Save Program (sticky footer) is disabled/dimmed until the name is non-empty; saving calls onBack.

Full-screen · Workout editor

VIEW · workout-editor

Edit a workout's exercises

Name field, two rest pickers, and a reorderable exercise list with a per-exercise action menu. Seeds from WK_SEEDS[workout.id] (falls back to push-a).

Header & rest
  • Editable workout name; a back chevron and a Save action (both call onBack).
  • Two RestPicker cards — Between sets (default 60s) and After exercise (default 90s) — each a stepper over a fixed ladder 15·30·45·60·75·90·120·150·180·240·300s with a dot indicator. Formats <60s as "Ns", else "m:ss".
Exercise cards
  • Each: drag grip, tappable name (→ Exercise detail in workoutCtx mode, carrying sets/reps/rest + superset role), chips ("{sets} sets", "{reps} reps", and an accent SS badge on a superset leader), and a ⋯ menu.
  • Drag-reorder: HTML5 drag; dragging dims the others (dashed). Drop recomputes the index (accounting for removal offset).
⋯ exercise menu (ex-menu)
  • Inline Sets stepper (min 1) + Rep-range text input (live-editing the card), then: Substitute exercise (→ picker in sub mode), Create / Remove Superset (→ ss-pick, or unlinks), Add exercise after (→ picker in add mode), and Remove exercise (red).

Exercise picker (full-screen)

VIEW · exercise-picker

Search + catalog to add / substitute

  • Back-header with a search field; muscle filter row + equipment filter row (same "Arms"→Biceps/Triceps mapping); 2-up muscle-tinted catalog grid. Empty → "No exercises found".
  • Three modes determine the select behaviour: sub (replace exercise i, header "Substitute" + "Replacing X"), add (insert after an index), ss-new (pick "Exercise B" to superset with the source). Selecting routes back to the editor and applies.

Supersets (the data model to get right)

ModelA superset is encoded as superset:true on the leader exercise; its partner is simply the next exercise in the array. There is no pair id. So isSSFirst = ex.superset===true and isSSSecond = prev.superset===true. Only one pair exists at a time in the prototype — creating a new superset clears all superset flags first.
  • Create (existing partner): ss-pick sheet shows the leader (badge A) then "pair with existing" rows (badge B). Choosing reorders the target to sit right after the leader and sets the leader's flag.
  • Create (new exercise): "Pick from library" → picker in ss-new → inserts the new exercise after the leader and flags the leader.
  • Render: a paired second card is preceded by a connector bar — two accent rules around a "⚡ SUPERSET" pill — and both cards get an accent-tinted border.
  • Remove: clears all superset flags.

Full-screen · Exercise detail

VIEW · exercise-detail

Overview · History (+ Workout)

Reusable detail screen (ExerciseDetail) launched two ways: from the Exercises library with showActions (adds the bottom action bar), or from the workout editor with workoutCtx (adds a leading Workout tab for inline set/rep/rest editing).

Tabs
TabWhenContents
Workoutonly with workoutCtxVideo, Sets stepper, Rep-range input, Rest picker (ladder 30…300s), Save Changes → writes back sets/reps/rest to the editor card.
OverviewalwaysCategory/Equipment/Level pill, demo video w/ play, description, accent Pro tip, Muscle activation (% bars + SVG body map), numbered Key takeaways.
HistoryalwaysPersonal bests (Est. 1RM, Max Weight, Max Reps, Max Volume) + 5 seeded sessions, each expandable to a per-set table with Est-1RM.
Computed values
  • Level (levelFor) is derived from equipment: Bodyweight/Machine/Smith → Beginner, Dumbbell/Cable → Intermediate, Barbell → Advanced.
  • Est. 1RM uses the Epley formula w × (1 + reps/30) (rounded to 0.25; reps≤1 returns the weight). PBs are the max across seeded sessions; bodyweight shows "BW".
  • Muscle activation + body map come from XD_INFO[id] when present, else a per-muscle default. Primary muscles fill accent, secondary fill blue, on a front/back SVG silhouette.
Superset mode
  • When workoutCtx.isSuperset, an A / B segmented control (SSSubTabs) sits atop Overview/History and swaps which exercise's info is shown (leader vs partner).
Action bar (showActions)
  • Add to Today's Workout and Add to a Plan. The latter opens a workout picker (active plan) → then an "Add as new exercise" / "Replace one" sheet (replace lists same-muscle alternatives).

Scenario checklist

  • Sub-tab pills switch bodies; the header + opens Add-plan from any sub-tab.
  • My Plans week strip seeds Mon/Tue/Thu/Fri training, Wed/Sat/Sun rest; order flips with calStart.
  • Tapping a training day → day menu; a rest day → assign sheet; assigning/removing updates the strip + nulls deleted workouts' days.
  • Programs: all 5 filter dropdowns work (incl. PROG_META-derived ones), Clear resets, "Added" badge on the active program.
  • Workouts library filters by chip + search; cards open the editor.
  • Exercises library: "Arms" chip surfaces Biceps + Triceps; equipment chips stack with muscle chips; cards open detail with the action bar.
  • Workout editor: rename, both rest pickers, drag-reorder, sets/rep edits via ⋯, substitute, add-after, remove.
  • Superset: create with existing partner (reorders) and with a new library pick; connector + SS badge render; remove unlinks; only one pair at a time.
  • Exercise detail: Workout tab only appears from the editor and writes back on Save; Overview body-map highlights correct muscles; History PBs use Epley; A/B toggle in superset mode.
  • Program editor: difficulty optional; week strip blocked (warning) until a workout exists; Save disabled until named.
  • create-workout: 12-icon grid, disabled until named, then jumps straight into the editor.
  • Keyword colouring: push/pull/leg/upper themed; unmatched names fall back to accent + dumbbell.
