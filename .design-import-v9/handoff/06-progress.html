Phase 6

Progress tab

The analytics surface. Two sub-tabs — Stats (consistency, strength, exercise trends, PRs) and Body (measurements, photos, and a live nutrition calculator). Several pieces are genuinely computed, not static — the trend charts, the measurement series, and the entire TDEE/macro engine recalculate from inputs.

Sourcecombined/progress.jsx (748 lines — window.ProgressTab). Reuses AuraSeg, LineChart, Stepper from ui.jsx and defines a local AxisChart (labelled SVG). Receives tab, onTab, dark, onDark, logStat from AuraRoot.

Structure

Header = "Progress" large title + a sub-tab pill row (Stats · Body). Body has a second segmented control (Measurements · Nutrition). One component holds everything; sub-views are functions returning the body.

Stats sub-tab

PROG · heatmap

Consistency heatmap

  • GitHub-style month grid with ‹ / › month nav (clamped to Jan–Jun 2026). Days are generated deterministically per month via a seeded PRNG (genMonth) so the demo is stable.
  • 5 intensity levels (HEAT_BG / HEAT_LABEL): Rest · Partial · Swapped · Completed · PR day; a legend sits below. In the native build, map these to real session outcomes.
PROG · lifetime

Lifetime tiles

A 3-col grid of six stat tiles: Sessions 142 · Sets 3,480 · Volume 418k · PRs 37 · Hours 184 · Week streak 18. Static seed.

PROG · performance

Strength Score / Strength Balance

  • Strength Score (265): a level band (Beginner→Elite by 100-pt ranges) with "% to next" and per-muscle score bars (Legs/Chest/Back/Arms/Shoulders, hue-coded).
  • Strength Balance (70%): flags the weakest muscle ("Shoulders weakest") with per-muscle balance bars.
  • Which card(s) render is gated by logStat (see note above).
PROG · split

This week · volume by muscle

Horizontal % bars for Chest/Back/Legs/Shoulders/Arms/Core (SPLIT seed).

PROG · trends

Exercise Trends (computed chart)

  • Pick an exercise (swap button → search sheet with muscle + equipment filters over 12 exercises), a metric (1RM / Max Weight / Max Reps / Max Volume), and a range (1M/3M/6M/1Y).
  • Data comes from EX_DATA (12-point generated series per exercise via _genEx); sliceEx slices to the range — and for 1M it interpolates the last month into 4 weekly points. The big number + delta badge compare last vs first of the visible window.
  • AxisChart is a from-scratch labelled SVG: "nice" Y-ticks (1/2/5×10ⁿ snapping, exactly 4 intervals), X labels from EX_XLABELS, gradient area + line + end dot. Reuse the tick algorithm to keep axes clean at any scale.
PROG · prs

Personal records

Category chips (Chest/Back/Legs/Shoulders/Arms) filter a PR list (trophy, name, date, weight × reps). PR_DATA seed.

Body · Measurements

BODY · measurements

Measurement tracking

  • Metric chips (first 7 of 9 defs) select the active measurement; a hero card shows current value, delta-from-first badge, an info button (→ how-to sheet), an AxisChart trend, and a 1M/3M/6M/1Y range toggle.
  • Current measurements grid (all 9: weight, body fat, chest, waist, arms, thighs, shoulders, neck, hips) — tapping any switches the active metric; unset values show "—" dimmed.
  • Log Measurements → the logm sheet. History list below.
Partial logging is realThe log sheet accepts only the fields you fill. On save, each provided value updates measure[k], pushes onto that key's series[k] (capped at 8 points), and prepends a History entry summarising the changed metrics. Empty fields are untouched. Preserve this partial-save behaviour.
BODY · photos

Photo progress

  • A segmented Side by side / Up · down layout toggle over two placeholder photos (date · weight). "Add comparison photo" flashes a toast. Replace placeholders with the user's real progress photos.

Body · Nutrition (the calculator)

BODY · nutrition

Live TDEE + macro engine

Everything recomputes from nut. This is the most formula-heavy screen — implement the math exactly.

ValueFormula
BMRMifflin-St Jeor: 10·wt + 6.25·height − 5·age + (Male ? +5 : −161)
TDEEround(BMR × ACT[activity]) — Sedentary 1.2 · Light 1.375 · Moderate 1.55 · Active 1.725 · Athlete 1.9
Target caloriesmax(1200, TDEE + GOAL_ADJ[goal]) — Lose fat −500 · Maintain 0 · Lean gain +200 · Gain muscle +400
BMIwt / (height/100)² → category <18.5 / <25 / <30 / else
Macrossplit % (MACRO_SPLIT) of calories: protein/carbs ÷4 kcal·g, fats ÷9; fiber = cal/1000 × 14
  • Inputs: a weight-trend mini chart + target badge; a Your details card (Edit → details sheet with Steppers + Sex/Activity); BMI & TDEE tiles; a Goal segmented; a Macro split chip row (Balanced / High carb / High protein / Keto).
  • The accent Daily target card shows total kcal, a proportional protein/carbs/fats bar, and the four macro gram values — all live.

Scenario checklist

  • Heatmap nav clamps Jan–Jun; cells render stable per-month intensities with a legend.
  • Performance cards obey logStat (Score / Balance / Both).
  • Exercise trends: swap exercise (filterable sheet), switch metric + range; 1M interpolates 4 weekly points; delta badge + nice Y-axis correct.
  • PR category chips filter the list.
  • Measurements: metric chips + grid switch the active metric; chart + delta update; range toggle works.
  • Log Measurements saves only filled fields, updates the value, extends the series (max 8), and adds a history row.
  • Photo layout toggles side/stack.
  • Nutrition: editing any detail (height/weight/age/sex/activity/goal/macro) recomputes BMR→TDEE→calories→macros + BMI category live, with the 1200-kcal floor.
