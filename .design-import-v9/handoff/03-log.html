Phase 3

Log tab

The home screen and the most state-heavy surface in the app. A single component (LogTab) renders any of six day-states for the selected date and hosts eleven sheets. Everything keys off one derived value — dayInfo(date).kind — computed from the program template plus per-day overrides. Get the derivation right and the rest falls out.

SourceEntire tab lives in combined/log.jsx (771 lines). It is handed tab, onTab, onStartWorkout, inProgress, dark, calStart from AuraRoot (see Chapter 2). All state is in-memory React state — nothing here persists to storage in the prototype; in the native build, workout logs and overrides must be durable.

Overview

Top to bottom the screen is: a nav bar (date + large title + calendar button), a week strip (7 day cells with status dots + week navigation), and a body that switches on the selected day's kind. Sheets slide up over everything; a toast and the resume banner float above the tab bar.

nav bar→ week strip→ body(kind)+ sheet overlay+ toast+ resume banner+ tab bar

Program & seed data

The default schedule is a Push·Pull·Legs program. Days with no entry are rest days.

DaySunMonTueWedThuFriSat
PLAN_BY_DOWrestPull ALeg ArestPush APull Brest
  • PROGRAM_NAME = "Push · Pull · Legs" — shown as the accent badge on every card state.
  • WK_DETAIL holds the five PPL workouts (push-a · pull-a · leg-a · push-b · pull-b) plus four library workouts for the other plans, each with { name, muscles, duration, ex:[[name, sets, repRange]…] }.
  • SEED_MISSED = 2026-06-23 · 2026-06-17 · 2026-06-11 — past planned days pre-marked as missed (so the timeline isn't a perfect streak).
  • OTHER_PLANS = HIIT Cardio (hiit-full, hiit-upper) and 5×5 Strength (5x5-a, 5x5-b) — surfaced inside the Switch sheet.
NativeThese are prototype seeds. Replace PLAN_BY_DOW with the user's actual active plan from the Plan tab, SEED_MISSED/logged days with real history, and the workout details with the program's real exercise lists. The shape is the contract; the values are placeholders.

How a day's state is derived — dayInfo(date)

This pure function is the brain of the tab. Given a date it returns { iso, dow, rel, workoutId, kind, ov }. The body, the week-dot, and the calendar dot all read kind. Evaluate in this exact order:

#StepResult
1Look up ov = overrides[iso] and base workoutId = PLAN_BY_DOW[dow].—
2If ov.type ∈ {switch, added, logged} → workoutId = ov.workoutId.workout swapped in
3If ov.type ∈ {rest, removed} → workoutId = null.workout cleared
4Compute rel = past / today / future by comparing to LOG_TODAY.—
5No workoutId → today ? 'rest-today' · (removed && today) ? 'empty-today' · else 'rest'.rest/empty
6ov.type === 'logged' → 'done'.completed
7rel === 'today' → 'today'; rel === 'future' → 'future'.planned
8Else (past): ov.type==='added' ? 'done' · SEED_MISSED.has(iso) ? 'missed' · else 'done'.history
SubtleStep 5 runs before the logged/today checks — so once a day has no workout (rest or removed) it can never be 'today'/'done' until a workout is re-assigned. And step 8 means a past planned day defaults to 'done' unless it's in SEED_MISSED — i.e. the app optimistically assumes past planned sessions happened. Preserve both quirks.

The seven internal kinds collapse to six rendered day-states (rest + rest-today share one renderer):

today· done· missed· future· rest / rest-today· empty-today

Override types (what gets written to overrides[iso])

typeShapeWritten byEffect on kind
rest{type:'rest'}menu "Make Rest Day", missed "Mark as Rest"→ rest / rest-today
removed{type:'removed'}menu "Remove from Today"; move-sourcetoday → empty-today; else rest
added{type:'added', workoutId}assignWorkout (non-past), move-targetassigns workout; past → done
logged{type:'logged', workoutId}assignWorkout (logpast/past), edit-log & quick-log save→ done (+ workoutLogs[iso])
edit{type:'edit', workoutId, exercises[]}Edit-workout sheet savekeeps today; exList uses custom exercises
switch{type:'switch', workoutId}handled defensively in dayInfoswaps workout (assignWorkout normalizes to added/logged)
exList(info)The exercise list shown is: if ov.type==='edit' use ov.exercises; else the workout's seed ex[] mapped to {name, sets, reps}. Edits are today-only and never touch the program.

Header & week bar

LOG · header

Nav bar + week strip

Nav bar
  • Title = "Today" if selected is today, else the full weekday (e.g. "Thursday"). Sub = uppercase "THURSDAY, JUN 25".
  • Calendar button (top-right) sets calMonth to the selected month and opens the calendar sheet.
Week strip
  • Range label e.g. "Jun 22 – 28" (collapses the second month name when the week doesn't cross a boundary). Week starts on calStart (Sun or Mon, from Profile).
  • ‹ / › step the selection by ±7 days. The › (next week) is disabled + dimmed when already on the current week (no future-week browsing past the live week start).
  • "Today ›" ghost button appears whenever the user isn't on the current week or isn't on today; tapping it jumps selected back to today.
  • 7 day cells: weekday letter + date; selected cell fills with --accent + white text. Status dot below: done=green · planned(today/future)=accent · rest=gray · missed=red · empty=transparent. On the selected cell the dot turns white.
EdgeTapping a week cell only changes selected — it does not change the week shown unless you cross into a new week via the arrows. The missed dot uses a softened red (color-mix --red 75%), distinct from the calendar legend's solid red.

The six day-states

LOG · today

Today — planned workout

The default live state. Accent PPL badge, a card with the workout name + "{n} exercises · {muscles}", a scrollable numbered exercise list, and the action stack.

Card header
  • Workout name (22px/800) + meta line. A ⋯ button (top-right of card) opens the manage menu.
  • Exercise rows (ExRows): numbered chip, name, "{sets} sets · {reps} reps". Max-height 168px, scrolls internally.
Actions
ButtonDoes
Start Workout (primary)onStartWorkout(meta.name||'Workout', false) → opens Active Workout overlay (Chapter 5).
Log past (gray, half)Opens Log-past sheet with showToday:true (so "Log Today's Workout" appears).
Switch (gray, half)Opens Switch sheet.
LOG · done

Completed

A logged or past-assumed-done session. Adds a green "Completed" badge + green check disc on the card.

  • Exercise list shows at full opacity (not dimmed).
  • View Log (primary) → View-log sheet. Edit Log (gray) → Edit-log sheet, pre-filled from workoutLogs[iso] if present, else blank rows sized to the plan.
NoteA past planned day with no explicit log still renders as done (step 8). Its View Log shows the planned set count with em-dash weights until the user edits real numbers in.
LOG · missed

Missed

A past planned day in SEED_MISSED with no logged override. Red "Missed" badge + red ✕ disc; the exercise list is dimmed to 55%.

  • Log Workout (primary) → picker in logpast mode for that date (records the session retroactively → flips to done).
  • Mark as Rest Day (gray) → writes {type:'rest'} and flashes — clears the missed flag.
LOG · future

Future — read-only

A planned day after today. Same card but with a gray "View only" chip; the exercise list is dimmed.

  • Only action is View Workout (gray) → flashes "Read-only preview" (no editing of future days from Log).
  • An accent info card explains "Future workouts are read-only" with a "Go to Plans →" link that calls onTab('plan').
RuleFuture days cannot be edited, switched, or logged from the Log tab — the only path to change them is the Plan tab. Don't add shortcuts that bypass this.
LOG · rest

Rest day

No workout scheduled (program rest, or user marked rest). Centered moon glyph card, "Rest Day", recovery copy.

  • Past/today rest: a "Did you train anyway?" section with Add a Workout (today → add sheet; past → picker in logpast mode) and, on today only, Log a Past Workout.
  • Future rest: no actions — just an info card ("Scheduled rest day. Enjoy the recovery…").
LOG · empty-today

Empty / unplanned (today)

Today after the planned workout was removed ({type:'removed'}). Dashed-border card, accent + glyph, "Nothing planned".

  • Same action pair as today-rest: Add a Workout → add sheet, and Log a Past Workout → log-past sheet.
  • Distinguished from rest by the dashed border + plus glyph (vs solid border + moon).

The eleven sheets

All sheets are the shared Sheet bottom-sheet (scrim + grabber + optional title/sub + scrolling body). Only one is open at a time via sheet={kind,…}. Tapping the scrim or ✕ closes (setSheet(null)).

SHEET · menu

Manage today's workout

Opened from the today-card ⋯. Five rows, all today-only (a footnote reinforces "won't affect your program").

RowAction

Make it a Rest Daywrites {type:'rest'}, closes, flashes "Marked as rest day".

SHEET · edit

Edit Workout (today only)

  • Each exercise row: name, rep-range, a stepper for set count (min 1), and a red trash to remove the exercise. Edits mutate a working copy held in sheet.exercises.
  • Save for Today writes an override carrying the custom exercises (type edit, or preserves logged/existing type if one was set), flashes "Workout updated for today".
EdgeThe save reuses the day's existing override type when present — editing a logged day stays logged, editing a fresh planned day becomes edit. Don't collapse this to a single type or you'll lose the completed flag.
SHEET · move

Move to Another Day

  • Lists the 7 days of the current week with each day's current workout/rest. The selected (source) day is disabled + dimmed and tagged "(today)".
  • Tapping a target: writes {type:'removed'} to the source iso and {type:'added', workoutId} to the target, closes, flashes "Moved to {Day}".
SHEET · switch-v2

Switch Workout (two levels)

Replace today's workout with another, for today only.

  • Level 1: the active PPL program's 5 workouts (section "Push · Pull · Legs (Active)"), then Other Plans (HIIT, 5×5) as drill-in rows, then a Workout Library "Browse Library" card.
  • Picking a program workout → assignWorkout(id,'switch') (resolves to an added override on today).
  • Level 2 (sheet.planId set): a back-header + that plan's workouts; picking one assigns it. "Browse Library" only flashes "Opening workout library…" (stub).
SHEET · add

Add a Workout (source picker)

  • Three source cards: From your programs and From Workout Library both open the picker in add mode; Empty Workout calls onStartWorkout('My Workout', true) → blank Active Workout.
  • Reachable from: today-rest/empty "Add a Workout", and the FAB → Start Workout on the Log tab.
SHEET · logpast

Log a Past Workout

Two questions: when and which.

  • When: an optional "Log Today's Workout" row (only if showToday, → opens quick-log), then radio rows for Yesterday and 2 days ago (selected state stored in sheet.date), then "Pick a date" → calendar in forLogPast mode.
  • Which workout: three source cards (From a Program / A Saved Workout / Build from Library) → all open the picker in logpast mode for the chosen date.
SHEET · pick

Workout picker

  • Lists the 5 PPL workouts (thumb, name, "{n} exercises · {muscles}"). Sub-title reflects the mode: "Logging to {date}" for logpast, "Added to today/{day}" for add.
  • Selecting runs assignWorkout(id, mode, date): mode logpast → logged; otherwise past dates → logged, present/future → added. It also moves selected to that date, closes, and flashes.
SHEET · cal

Calendar (month)

  • Month grid with status dots per day (done/planned/rest, hidden for future days). Today is the accent disc; the selected non-today day gets an accent ring.
  • ‹ / › change month; › is disabled when it would pass the current month (no browsing into the future). ✕ closes.
  • Default mode: tapping a non-future day sets selected + closes; a "Go to Today" button jumps to today. forLogPast mode: tapping a day routes back to the log-past sheet with that date instead, and the "Go to Today" button is hidden.
  • Future days are dimmed and non-interactive. A legend (Completed / Planned / Rest) sits under the grid.
SHEET · view-log

View Log (read-only)

  • Per exercise: a Set / Weight / Reps table. Uses workoutLogs[iso] if present, else synthesizes rows from the plan with em-dash weights and planned reps.
  • Footer: Edit Log → edit-log (deep-copies the data first); Close.
SHEET · edit-log

Edit Log

  • Editable Set/Weight/Reps inputs per set, a per-row remove (minus-circle), and an Add Set button per exercise. All mutations run on a deep-copied working array in sheet.exercises.
  • Save Log writes workoutLogs[iso] = {…, exercises} and sets the day's override to logged, flashes "Log saved".
SHEET · log-quick

Quick log (with time)

  • Like edit-log but with a time picker card up top ("When did you train?", defaults to now). Inputs are placeholdered kg / reps.
  • Save Workout resolves the day's workoutId (override or PLAN_BY_DOW), writes workoutLogs[iso] = {time, exercises} + a logged override, flashes "Workout logged!".
  • Entered from the "Log Today's Workout" row inside the log-past sheet.

Toast · resume banner · FAB

  • Toast: flash(msg) sets toast and auto-clears after 1900ms (guarded so a newer toast isn't wiped by an older timer). Pill centered above the tab bar, z-index:90.
  • Resume banner: rendered here when props.inProgress — full spec in Chapter 2. Tapping calls the same startWorkout.
  • FAB routing (Log): onAction('workout') opens the add sheet; measure/photo → onTab('progress'). (Per-tab routing table in Chapter 2.)

Scenario checklist

  • Each weekday shows the correct seed: Mon Pull A · Tue Leg A · Thu Push A · Fri Pull B; Sun/Wed/Sat rest.
  • Past planned days read as done except the three SEED_MISSED dates, which read missed.
  • Today → Start Workout opens the overlay; Log past / Switch open their sheets.
  • Menu: Edit (today-only, preserves logged), Move (source→removed, target→added), Switch (2 levels), Rest, Remove (→ empty-today).
  • Removing today's workout shows the dashed "Nothing planned" card; marking rest shows the moon card.
  • Missed day → Log Workout flips it to done; Mark as Rest clears the missed flag.
  • Future day is read-only — only View Workout + the "Go to Plans" link; no edit/switch/log paths.
  • Log-past: pick a date (yesterday / 2 days ago / calendar), pick a workout → day becomes done at that date.
  • Quick-log captures a time; View Log / Edit Log round-trip the set data and keep the day done.
  • Calendar respects forLogPast (routes back to log-past) vs default (selects the day); never lets you go past the current month.
  • Week strip: next-week arrow disabled on the current week; "Today ›" appears whenever off-today; dots match each day's kind.
  • Week starts on Sunday or Monday per calStart from Profile.
  • Every mutating action fires a toast that auto-clears.
