Foundation

The design system, shared component primitives and the global state model. Everything else in the app is assembled from these pieces — build this layer first and the tabs go quickly.

Typography & scale

  • UI / display: Plus Jakarta Sans 400–800. Geometric, clean. On native you may substitute SF Pro.
  • Mono / captions: JetBrains Mono — used only for placeholder captions & code-like tags.
  • Numeric data (weights, timers, stats): display font with font-variant-numeric: tabular-nums and letter-spacing:-0.03em — class .stat-num. Keep this on native; it stops digits jittering as timers count.

Spacing · radius · shadow

  • Spacing base-4: 4 · 8 · 12 · 16 · 20 · 24 · 32 · 40 (--s1…--s10). Screen horizontal padding = 20px (.pad); list/scroll content reserves 110px bottom for the floating tab bar (.pad-b).
  • Radius: xs 8 · sm 12 · md 16 · lg 22 · xl 28 · pill 999. Cards use lg/xl; lists & fields md/sm; chips/toggles pill.
  • Shadow: --shadow-sm (cards, rows) and --shadow (elevated/hover). Sheets use a strong upward shadow.

Device frame

Everything renders inside .phone — a fixed 393 × 852 iPhone with 44px corner radius, clipped overflow. The whole frame is scaled to fit the viewport by a small fit() routine in Aura App.html (never resized).

  • Status bar 54px: time left, signal/wifi/battery right, Dynamic Island (124×36) centred. Auto-rendered by styles/icons.js.
  • Nav bar (.navbar, optionally .bordered): large title or back-button row.
  • Home indicator pill 134×5 at the bottom; the tab bar floats above it.
NativeThis frame is a prototype device; on a real build it maps to the OS safe-area insets. The 54/84px values translate to the system status bar & home-indicator safe areas.

Icon set

A single Icon component renders ~50 named glyphs from a path table (workout/icons.jsx, shared set in styles/icons.js). All are 24px viewBox, stroke ~1.7, round caps/joins, currentColor. Map each to its SF Symbol on iOS.

Shared components

Defined in combined/ui.jsx (and small local copies in tab files). Build these as reusable primitives — they appear on nearly every screen.

CMP · Sheet

Bottom sheet

Modal that slides up from the bottom over a dimmed scrim. The workhorse container for every picker, menu and form in the app.

Anatomy
  • Tap-dismissable scrim (--scrim) + a card pinned to the bottom, radius 28 top corners, max-height passed per sheet (48–93%).
  • Grabber pill (38×5) at top; optional title + subtitle row with an x close button.
  • Body scrolls internally; 28px bottom padding. Content never pushes the device frame.
CMP · Row

Grouped list row

iOS settings-style row inside a .list card: 30px coloured icon tile, title + optional subtitle, trailing value / chevron / control. Hairline separator inset 16px from the left.

CMP · Controls

Seg · Toggle · Stepper · Chips

  • Seg (segmented control): equal pills in a fill track; active pill = surface + shadow. Options can be string or {v,l}.
  • Toggle: 51×31 iOS switch; ON = green track, knob slides 20px.
  • Stepper: − / value / + with an optional fmt formatter (e.g. fmtRest renders seconds as "1 min 30 s"). The + button is accent-tinted.
  • Chip / filter: pill, fill bg; active = accent (or muscle-coloured in libraries). Filter rows scroll horizontally with hidden scrollbars.
CMP · LineChart / AxisChart

Charts

  • LineChart: lightweight SVG area+line, gradient fill, dot on the last point. Used in nutrition & small trends.
  • AxisChart (Progress): adds 4 "nice" Y gridlines/labels (auto-stepped to 1/2/5 multiples) and X labels. Used for measurement & exercise trends.
AssetThe .ph placeholder (striped box + mono caption) marks every spot needing a real image/video — exercise demos, body map, progress photos, program cover art.

Theming

Dark mode is a single boolean held in AuraRoot and persisted to localStorage["aura_dark"]. The root sets data-theme on every .phone element. Because some sub-views (Plan, Active Workout) hard-code data-theme="light", the shell runs a MutationObserver that re-applies the chosen theme whenever new phone surfaces mount.

EdgeOn native this is trivial (one environment value), but note the intent: dark mode must apply globally and immediately, including to the Active Workout overlay and any pushed sub-screen — not just the tab that toggled it.

State & persistence model

The prototype keeps state in component memory and mirrors a few keys to localStorage. On native, map these to your store / user-defaults / DB. Keys used:

KeyHoldsOwner
aura_dark"1"/"0" dark modeshell.jsx
aura_calstart"Sun"/"Mon" week startshell.jsx (set in Profile)
aura_logstat"Score"/"Balance"/"Both" — which card shows on Progressshell.jsx (set in Profile)
aura_wkActive workout object (versioned, schema v5)workout/app.jsx
aura_elapsedWorkout timer secondsworkout/app.jsx
aura_pillRest-pill {x,y} positionworkout/app.jsx
  • Cross-tab settings flow down as props from AuraRoot and changes flow up via callbacks (e.g. Profile's week-start updates the Log calendar live).
  • Schema versioning: the active workout is tagged version. On load, if the saved version ≠ current, the stored copy is discarded and fresh seed used — so a model change never crashes a restored session. Preserve this guard on native (migration or discard).
  • In-tab state (Log overrides, Progress measurements, Plan schedule) is local to each tab in the prototype and resets on reload. On native these become persisted records.
PrincipleEdits to a workout are held live and only committed on an explicit Save — see the Active Workout and Log chapters for the "today-only vs permanent" scope rule that governs what a save touches.
