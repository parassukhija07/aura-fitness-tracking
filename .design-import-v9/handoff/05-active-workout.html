Phase 5

Active Workout

The hero flow — a full-screen takeover (no tab bar) that runs a live training session. Six views, a draggable rest timer, set-type tagging, supersets, auto-rest logic, celebration overlays and a summary. The interaction rules here are the soul of the app; this chapter specifies every threshold and side-effect.

Sourceworkout/app.jsx (810 lines — controller, overview, empty mode, rest pill, modals, summary), workout/exercise.jsx (set-logging view), workout/superset.jsx (round-based pair view), workout/data.jsx (the seeded "Push Day A" + set types). Mounted by AuraRoot when workoutOpen — see Chapter 2.

The flow

The controller (WorkoutApp) holds the whole session and renders one of six views by a view string (plus an "empty" branch of overview). Logging screens push from overview and return to it.

overview⇄ exercise· superset→ summary| also minimized· empty overview (0 exercises)

Launch & restore

  • Props from the shell: onExit, onMinimize, workoutName, emptyMode.
  • emptyMode === true → starts a blank workout { name: workoutName||'My Workout', program:'Free Workout', exercises: [] } → renders the empty overview.
  • Otherwise → clone(WORKOUT) (seeded "Push Day A", 6 exercises), then attempts a restore from localStorage 'aura_wk'.
Schema-gated restoreThe saved blob carries a version. On load it's compared to WORKOUT.version (currently 5). On mismatch the key is cleared and fresh seed data is used. On match, it restores per-exercise sets, completed, note, pulley (matched by exercise id at the same index). Keep this guard — it prevents stale shapes from crashing the logger after a data-model change.

Data model (workout/data.jsx)

  • workout = { version, name, program, exercises[] }.
  • exercise = { id, name, muscle, groups[], equipment, isCable, repRange, planned, lastPR{weight,reps,date}, target{weight,reps,note}, history[], warmup[{reps,pct}], hint, pulley, sets[], superset?, completed?, note? }.
  • set = { weight, reps, done, type, note? }. weight/reps are strings (empty = unentered).
  • SET_TYPES: normal · warmup(W) · drop(D, purple) · restpause(R, blue) · failure(F, red) · partials(P, green). The picker excludes warmup.
  • SUB_OPTIONS (substitute list) and ADD_OPTIONS (add-exercise list) are fixed seed arrays.
planned vs setsplanned is the prescribed set count; sets.length can exceed it (the user added sets). history[] aligns by index — sets beyond planned surface an "extra sets last session" hint. Targets/PRs come from the exercise, not computed live.

The six views

WK · overview

Overview

The session hub: list of exercise cards with progress, plus add/finish.

  • Header: End (red → end modal) · centered workout name + running duration (mm:ss, accent) · Minimise (calls onMinimize, or local minimized view as fallback).
  • Progress: "{done}/{total} sets" + program name, and a fill bar.
  • Exercise cards: drag grip (reorder), tappable name (→ exercise detail), "{sets} sets · {repRange} reps · {equipment}", per-card mini progress bar, green check + dim-to-done when all sets logged. Tapping the card body opens the exercise (or the superset view if it's part of a pair). A ⋯ opens the overview exercise menu.
  • Superset cards render a "⚡ SUPERSET" connector above the second card and an accent border on both; tapping either opens the superset view at the leader.
  • Footer: Add Exercise (→ add modal) and Finish Workout (→ summary).
WK · empty

Empty / build-as-you-go

Shown whenever wk.exercises.length === 0 (empty-mode launch, or after removing every exercise).

  • Hero ("Build your workout"), a search button (→ add modal), Quick add by muscle chips (Chest/Back/Legs/Shoulders/Arms/Core, each its own colour), an equipment filter row, and a catalog grid.
  • With a muscle selected → that muscle's exercises filtered by equipment; otherwise a Suggested list. Tapping a card adds it and jumps straight into its exercise view (addExAndStart).
WK · exercise

Exercise logging

The detailed set logger (ExerciseView).

  • Header: back, "Exercise {n}" + name, ⋯ menu.
  • Demo video w/ play; tapping the title opens exercise detail. Equipment + muscle-group chips.
  • Cable exercises (isCable) show a Pulley setup card with a Single / Double segmented control (persisted to ex.pulley).
  • Last PR + Today's target mini-cards (target highlighted accent, with its note).
  • Warm-up protocol — collapsible, only rendered when warmup.length > 0 (in the seed, only the first two exercises have warm-ups; bench gets the full ramp, incline gets 2 sets). Lists "W{n} · {reps} reps · {pct}". Open by default.
  • Form tip card (the exercise hint).
  • Working sets: "{done}/{len} done" badge, progression bar, set rows (see Set logging), Add set. A blue hint surfaces any extra sets logged last session. An Exercise notes textarea (persisted to ex.note).
  • Complete Exercise (primary) — see Complete logic.
WK · superset

Superset logging (round-based)

SupersetView logs a pair (A = leader, B = ssIdx+1) as rounds instead of separate set lists.

  • Header shows an accent "⚡ Superset" title + ⋯. Progress = combined "{done}/{total} sets" and a "{rounds} rounds" pill (rounds = max(A.sets, B.sets)).
  • Two ExMeta cards (A accent, B blue) each with PR + target. Then one round card per round: "Round {n}", a 0/2 → 2/2 progress, and an A row + B row (tappable names → detail).
  • Add Round appends a set to both exercises. Two notes textareas (A, B). Complete Superset finalises both.
WK · minimized

Minimised (local fallback)

  • A dimmed Log-tab mock behind a resume banner showing the workout name + running timer + "Tap to resume" (→ overview) and an ✕ (→ end modal).
  • Note: in the real app the shell's onMinimize intercepts first (returning to the live Log tab with its resume banner — see Ch.2). This local minimized view is the standalone-prototype fallback when no onMinimize is wired.
WK · summary

Summary

  • Gradient hero ("🔥 Workout Complete" + name · program). Stat grid: Duration (elapsed), Sets (done), Volume (Σ weight×reps of done sets, kg), New PRs (exercises with a set beating lastPR.weight).
  • If PRs > 0, an accent banner ("{n} new personal record(s)! Logged to your Progress tab."). Per-exercise recap rows (done sets · volume). A Session notes textarea.
  • Save Workout → resetAll (clears storage, calls onExit). Back to workout → returns to overview. The workout timer is frozen here.

Interaction rules (implement exactly)

RULE · set rows

Set logging

  • Set-number button → opens the set-type picker. For a non-normal type the button shows the type's letter in its colour instead of the number.
  • Weight / reps are free-text decimal inputs (placeholder shows the matching history value when present).
  • Auto-finish: on blur, if both weight and reps are non-empty, the set auto-marks done. The ✓ check toggles done manually too.
  • Trash deletes the set. A history line (last session's weight×reps) sits under each row when the showHistory tweak is on and history exists.
RULE · rest

Don't over-normaliseThe durations are intentionally mixed: single-exercise completion is 90s, superset set-rest is 60s, set/add-rest follows the tweak. Replicate the exact values, not a single global.
RULE · celebrations

Celebration overlay (~2.4s)

ConditionOverlay
set weight > lastPR.weight🏆 New PR!
set reps > target.reps and weight ≥ target.weight🔥 Extra reps!
Complete Exercise💪 Exercise done

Remove superset pairing✂️ Superset split

PR and Extra-reps are checked on the same done toggle (PR takes precedence — it's an else if). All auto-dismiss after 2400ms; tapping is not required.

RULE · complete

Complete Exercise / Superset

  • Removes empty sets (weight==='' && reps==='' && !done), marks every filled set done, sets completed=true.
  • Fires the completion celebration, starts the corresponding rest (90s exercise / restDuration superset), and returns to overview.
  • Superset completion does this for both A and B.
RULE · rest pill

Floating rest pill

  • Appears whenever rest.active; floats over every logging view at pos. Draggable anywhere (pointer events), clamped within the phone (x 8…193, y 60…782), and the position persists to aura_pill.
  • Conic-gradient ring shows the remaining fraction; center shows "REST" + mm:ss. Controls: +15s, pause/play (toggles running), ✕ dismiss (sets active:false).
RULE · supersets

Superset model & menu

  • Same encoding as the Plan editor: superset:true on the leader; partner is the next exercise. One pair at a time — creating clears all flags first and reorders the chosen B to sit right after A.
  • Create via the exercise menu → ss-pick: pick an existing exercise (B) or "Add new exercise" (adds from ADD_OPTIONS, then links). Remove → remove-ss confirmation showing each side's logged-set count; splitting keeps all logged data.

Tweaks (prototype-only)

  • Open on (startView): overview / superset / exercise. Dark mode. Show history (the under-row last-session values). Default rest slider (30–180s, default 90).
  • These drive the demo's initial state and the rest default; they're not user-facing settings in the shipped app (the real rest default lives in Profile → Workout). Treat as authoring knobs.

Scenario checklist

  • Seed launch shows "Push Day A" mid-session (timer ~24:47, some sets pre-logged); restore survives reload but resets on a version bump.
  • Entering weight+reps and blurring auto-checks the set; the ✓ also toggles manually; trash removes a set.
  • Marking a non-final set done starts the tweak-length rest; the final set does not; adding a set starts rest.
  • Complete Exercise drops empty sets, marks filled done, fires 💪, starts 90s rest, returns to overview.
  • Beating lastPR.weight → 🏆; exceeding target reps at ≥ target weight → 🔥; both auto-dismiss ~2.4s.
  • Rest pill drags anywhere, persists, +15s / pause / dismiss all work; ring reflects remaining time.
  • Cable exercises show the pulley segmented control and persist the choice.
  • Warm-up protocol appears only on exercises that have one (bench full ramp, incline 2 sets) and is collapsible.
  • Set-type picker tags a set (D/R/F/P colour); the set-number button reflects it.
  • Create superset (existing or new B) pairs & reorders; superset view logs by rounds; Add Round adds to both; 60s rest per side; Complete finalises both; Remove splits but keeps data.
  • Add Exercise (overview/empty) inserts from suggestions; empty-mode card-tap adds and jumps into the logger.
  • End → Finish&Save shows summary (duration frozen, volume + PR count correct); Discard exits; Minimise returns to Log with the resume banner.
